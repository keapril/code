# 本地開發 Secrets 設定指南

## 問題
整理後執行 `streamlit run app.py` 出現錯誤：
```
Firebase 初始化失敗: No secrets.toml file
```

## 解決方法

### 方法一：建立本地 secrets.toml（推薦）

1. **複製範例檔案**
```bash
cd config/.streamlit
copy secrets.toml.example secrets.toml
```

2. **編輯 secrets.toml**
填入真實的 Firebase 設定：
```toml
[firebase]
type = "service_account"
project_id = "你的專案ID"
private_key_id = "..."
private_key = "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
client_email = "..."
client_id = "..."
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "..."
```

3. **重新啟動 Streamlit**
```bash
cd src
streamlit run app.py
```

### 方法二：在專案根目錄建立 .streamlit/secrets.toml

如果方法一不行，可以在專案根目錄建立：

```bash
# 在 APP｜院內碼/ 目錄下
mkdir .streamlit
copy config\.streamlit\secrets.toml.example .streamlit\secrets.toml
```

然後編輯 `.streamlit/secrets.toml` 填入真實資料。

### 方法三：暫時跳過 Firebase（僅測試介面）

如果只是想測試介面，可以暫時註解掉 Firebase 相關程式碼。

## 注意事項

⚠️ **secrets.toml 已加入 .gitignore**
- 這個檔案包含敏感資料
- 不會被上傳到 GitHub
- 只存在於本地

## Firebase Key 位置

你的 Firebase key 可能在：
- `--開發系統資料--/` 資料夾
- 或之前的專案設定中
